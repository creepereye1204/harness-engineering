# CONSTITUTION

## 1. 주석 없음
주석을 작성하지 않는다. 이름이 설명한다.
함수명, 변수명, 타입이 의도를 충분히 전달하지 못하면 이름을 고친다.
주석으로 해결하지 않는다.

## 2. 레거시 스타일 없음
Java: raw type, 명시적 for-loop(stream으로 대체 가능 시), XML 설정(애노테이션으로 대체 가능 시) 금지.
React: 클래스 컴포넌트, var, callback 패턴(hook으로 대체 가능 시) 금지.
현재 언어/프레임워크의 관용적 최신 방식만 사용한다.

## 3. 유지보수 가능한 구조
하나의 함수/클래스는 하나의 책임만 진다.
깊은 중첩(3단 이상 if/try)은 early return 또는 추출로 평탄화한다.
중복이 3번 이상 반복되면 추출한다. 그 전에는 추출하지 않는다.

## 4. 네 가지 절대 규칙
**간결하게** — 동작에 필요한 최소한만 작성한다. 방어 코드, 미래 대비 추상화, 사용되지 않는 옵션을 추가하지 않는다.
**안전하게** — 입력 검증은 시스템 경계(API 진입점)에서만. SQL 인젝션, XSS, IDOR 등 OWASP Top 10을 생성 시점에 차단한다.
**문서 없이 기능 추가 없음** — product-specs에 없는 기능은 구현하지 않는다. 스펙이 없으면 먼저 사용자에게 확인한다.
**테스트 속임 없음** — @Disabled, 하드코딩된 반환값, 실제 동작을 우회하는 mock으로 테스트를 통과시키지 않는다. 테스트가 실패하면 코드를 고친다.
