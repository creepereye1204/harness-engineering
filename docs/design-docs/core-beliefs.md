# design-docs/core-beliefs.md — UI/UX 핵심 원칙

## 1. 콘텐츠 중심
- 글과 댓글이 주인공. UI는 콘텐츠를 방해하지 않는다
- 불필요한 여백, 장식 요소 최소화
- 목록에서 최대한 많은 글을 한 눈에 볼 수 있도록

## 2. 빠른 피드백
- 모든 사용자 액션(추천, 댓글, 신고)은 즉시 시각적 반응
- 로딩 상태 항상 표시 (스켈레톤 UI 사용)
- 에러 발생 시 토스트 메시지로 즉시 안내

## 3. 일관성
- 동일한 액션은 항상 동일한 UI 패턴 사용
- 컴포넌트 재사용 최우선
- Claude Code는 새 컴포넌트 만들기 전에 components.md 먼저 확인

## 4. 접근성
- 색상만으로 정보를 전달하지 않는다 (아이콘/텍스트 병행)
- 키보드 네비게이션 지원
- 다크 모드에서도 동일한 가독성 보장

## 5. 모바일 우선
- 모든 컴포넌트는 모바일 기준으로 먼저 설계
- 브레이크포인트: mobile(~768px) / tablet(768~1024px) / desktop(1024px~)
