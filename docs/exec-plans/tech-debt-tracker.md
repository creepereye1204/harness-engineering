# docs/exec-plans/tech-debt-tracker.md — 기술 부채 트래커

## 개요
빠른 개발을 위해 임시로 처리한 항목들을 추적.
새 기능 작업 전 이 목록을 확인하고 HIGH 항목부터 해소할 것.

## 트래커

| 항목 | 현재 구현 | 목표 구현 | 우선순위 | 발생 시점 | 상태 |
|------|-----------|-----------|----------|-----------|------|
| 조회수 중복 방지 | DB post_view_logs 테이블 | Redis TTL 기반 | HIGH | Phase 4 설계 시 | 미해결 |
| 전체 검색 최적화 | LIKE 쿼리 | MySQL Full-text Search | MEDIUM | Phase 4 설계 시 | 미해결 |
| 이메일 발송 비동기 처리 | 트랜잭션 밖 동기 처리 | Spring @Async 또는 이벤트 기반 | MEDIUM | Phase 2 설계 시 | 미해결 |
| 인기글 점수 조회수 반영 | 10분 배치 갱신 | 실시간 또는 단축 배치 | MEDIUM | Phase 4 설계 시 | 미해결 |
| 테스트 커버리지 | 핵심 로직만 | 전체 80% 이상 | MEDIUM | 프로젝트 시작 시 | 미해결 |
| 첨부 파일 CDN | 로컬 Docker 볼륨 | S3 또는 Cloudflare R2 | LOW | Phase 6 설계 시 | 미해결 |

## 규칙
- 새 항목 발견 시 즉시 이 파일에 추가 (발생 시점 명시)
- 해소 완료 시 상태를 "완료 (YYYY-MM-DD)"로 변경
- 구현 착수 전부터 이미 등록된 HIGH 항목은 카운트에서 제외한다. 구현 중 새로 발견된 HIGH 항목이 3개 이상이면 기능 추가 중단 후 해소 우선
