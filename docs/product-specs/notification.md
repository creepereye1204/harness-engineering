# notification.md — 알림 기능

## 기능 설명
사용자 활동 기반 실시간 알림 기능.
SSE(Server-Sent Events)로 단방향 실시간 푸시.
알림 목록 조회, 읽음 처리, 설정 관리 포함.

## 알림 발생 조건

| 알림 유형 | 발생 조건 | 수신자 |
|-----------|-----------|--------|
| COMMENT | 내 게시글에 댓글 작성 | 게시글 작성자 |
| REPLY_ON_POST | 내 게시글에 대댓글 작성 | 게시글 작성자 |
| REPLY_ON_COMMENT | 내 댓글에 대댓글 작성 | 댓글 작성자 |
| VOTE_POST | 내 게시글 추천 | 게시글 작성자 |
| VOTE_COMMENT | 내 댓글 추천 | 댓글 작성자 |
| REPORT_RESULT | 내 신고 처리 완료 | 신고자 |

### 알림 제외 조건
- 본인 행동에 의한 알림 발생 X
- 추천 알림은 동일 사용자가 같은 글에 중복 발생 X
- 차단된 사용자의 행동은 알림 발생 X

## 사용자 시나리오

### 실시간 알림 수신
1. 로그인 시 SSE 연결 자동 수립
2. 알림 발생 시 헤더 알림 아이콘에 뱃지 표시
3. 알림 클릭 시 해당 게시글/댓글로 이동
4. 페이지 이동/새로고침 시 SSE 재연결

### 알림 목록 조회
1. 헤더 알림 아이콘 클릭 → 드롭다운으로 최근 10개 표시
2. "전체보기" 클릭 → 알림 전체 목록 페이지
3. 읽지 않은 알림은 강조 표시
4. 알림 클릭 시 자동 읽음 처리 + 해당 페이지로 이동

### 읽음 처리
1. 개별 알림 클릭 시 읽음 처리
2. "모두 읽음" 버튼으로 전체 읽음 처리
3. 읽은 알림은 30일 후 자동 삭제

### 알림 설정
1. 알림 유형별 수신 여부 개별 설정 가능
2. 설정 변경은 즉시 반영

## 상세 요구사항

### SSE 연결
- 로그인 사용자만 SSE 연결
- 연결 타임아웃: 30분 (이후 클라이언트가 재연결)
- 하트비트: 15초 간격으로 ping 전송 (연결 유지)
- 비로그인 전환 시 연결 종료

### 알림 보관
- 최대 100개 보관 (초과 시 오래된 것부터 삭제)
- 읽은 알림: 30일 후 자동 삭제 (created_at 기준)
- 읽지 않은 알림: 90일 후 자동 삭제 (created_at 기준)

### 추천 알림 묶음 처리
- 1시간 내 동일 게시글/댓글 추천이 여러 개면 묶어서 발송
  - 예: "OOO 외 4명이 회원님의 게시글을 추천했습니다"

## API 목록

| 메서드 | 엔드포인트 | 설명 | 인증 |
|--------|-----------|------|------|
| GET | /api/v1/notifications/subscribe | SSE 연결 | 필요 |
| GET | /api/v1/notifications | 알림 목록 조회 | 필요 |
| PATCH | /api/v1/notifications/{notificationId}/read | 알림 읽음 처리 | 필요 |
| PATCH | /api/v1/notifications/read-all | 전체 읽음 처리 | 필요 |
| DELETE | /api/v1/notifications/{notificationId} | 알림 삭제 | 필요 |
| GET | /api/v1/notifications/settings | 알림 설정 조회 | 필요 |
| PATCH | /api/v1/notifications/settings | 알림 설정 수정 | 필요 |

### SSE 이벤트 포맷
```json
{
  "notificationId": "uuid",
  "type": "COMMENT",
  "message": "OOO님이 회원님의 게시글에 댓글을 달았습니다",
  "targetUrl": "/b/free/posts/123",
  "isRead": false,
  "createdAt": "2024-01-01T00:00:00"
}
```

## 예외 처리

| 상황 | HTTP 상태 | 에러 코드 | 메시지 |
|------|-----------|-----------|--------|
| 비로그인 SSE 요청 | 401 | AUTH_REQUIRED | "로그인이 필요합니다" |
| 존재하지 않는 알림 | 404 | NOTIFICATION_NOT_FOUND | "존재하지 않는 알림입니다" |
| 타인 알림 접근 | 403 | NOTIFICATION_FORBIDDEN | "접근 권한이 없습니다" |
