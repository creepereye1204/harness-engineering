# report.md — 신고 기능

## 기능 설명
게시글, 댓글, 사용자를 신고하는 기능.
신고 접수 → 관리자 검토 → 처리 결과 알림 순으로 진행.
자동 블라인드 기준은 service_settings 테이블에서 조회.

## 신고 대상 및 사유

### 신고 대상
- 게시글
- 댓글/대댓글
- 사용자

### 신고 사유
| 코드 | 사유 |
|------|------|
| SPAM | 스팸/도배 |
| OBSCENE | 음란/성인 콘텐츠 |
| ABUSE | 욕설/비하/혐오 |
| ILLEGAL | 불법 정보 |
| PRIVACY | 개인정보 노출 |
| COPYRIGHT | 저작권 침해 |
| OTHER | 기타 (직접 입력) |

## 사용자 시나리오

### 신고 접수
1. 게시글/댓글의 더보기 메뉴에서 "신고" 클릭
2. 신고 사유 선택
3. OTHER 선택 시 상세 내용 직접 입력 (최대 500자)
4. 신고 완료 → "신고가 접수되었습니다" 토스트 메시지
5. 동일 대상에 중복 신고 불가

### 자동 블라인드
1. 동일 대상 신고 누적 시 자동 블라인드 발동
2. 기준값: service_settings의 auto_blind_threshold (기본값: 5)
3. 자동 블라인드된 게시글은 관리자 검토 후 최종 처리

### 신고 처리 (관리자)
1. 관리자 페이지에서 신고 목록 확인
2. 신고 내용 검토
3. 처리 결과 선택:
   - 콘텐츠 삭제
   - 사용자 경고 (users.warning_count +1, 자동 정지 체크)
   - 사용자 일시정지
   - 사용자 영구정지
   - 신고 기각
4. 처리 완료 → 신고자에게 알림 발송

### 경고 누적 자동 정지
- 경고 부여 시 users.warning_count +1
- warning_count가 service_settings의 auto_suspend_warning_count(기본값: 3) 이상이면
  자동으로 일시정지 처리
- 정지 기간: service_settings의 auto_suspend_days(기본값: 7일)
- 자동 정지 시 해당 사용자 즉시 로그아웃 처리

## 상세 요구사항

### 신고 제한
- 로그인 사용자만 신고 가능
- 본인 게시글/댓글 신고 불가
- 동일 대상에 중복 신고 불가
- 하루 최대 신고 횟수: 20회 (어뷰징 방지)

### 사용자 제재
| 제재 유형 | 내용 |
|-----------|------|
| 경고 | warning_count +1, 누적 기준 초과 시 자동 정지 |
| 일시정지 | 기간 내 로그인/글쓰기 불가, 즉시 로그아웃 |
| 영구정지 | 로그인 불가, soft delete 처리 |

## API 목록

| 메서드 | 엔드포인트 | 설명 | 인증 |
|--------|-----------|------|------|
| POST | /api/v1/reports | 신고 접수 | 필요 |
| GET | /api/v1/admin/reports | 신고 목록 조회 | 관리자 |
| GET | /api/v1/admin/reports/{reportId} | 신고 상세 조회 | 관리자 |
| PATCH | /api/v1/admin/reports/{reportId}/process | 신고 처리 | 관리자 |

## 예외 처리

| 상황 | HTTP 상태 | 에러 코드 | 메시지 |
|------|-----------|-----------|--------|
| 본인 신고 시도 | 403 | REPORT_SELF_NOT_ALLOWED | "본인 게시글/댓글은 신고할 수 없습니다" |
| 중복 신고 | 409 | REPORT_DUPLICATE | "이미 신고한 게시글/댓글입니다" |
| 일일 한도 초과 | 429 | REPORT_DAILY_LIMIT | "오늘 신고 가능 횟수를 초과했습니다" |
| 존재하지 않는 대상 | 404 | REPORT_TARGET_NOT_FOUND | "존재하지 않는 게시글/댓글입니다" |
| 비로그인 신고 | 401 | AUTH_REQUIRED | "로그인이 필요합니다" |
