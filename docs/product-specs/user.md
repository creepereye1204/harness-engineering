# user.md — 회원가입/로그인/프로필

## 기능 설명
사용자 인증 및 프로필 관리 기능.
JWT 기반 로그인, 회원가입, 이메일 인증, 내 정보 조회/수정, 회원 탈퇴를 포함.

## 사용자 시나리오

### 회원가입
1. 이메일, 비밀번호, 닉네임 입력
2. 이메일 중복 확인 (탈퇴 계정 포함 전체 레코드 대상)
3. 닉네임 중복 확인 (deleted_at IS NULL인 활성 계정만 대상)
4. 가입 완료 → 인증 이메일 발송
5. 이메일 내 링크 클릭 → 인증 완료
6. 로그인 페이지로 이동

### 로그인
1. 이메일 + 비밀번호 입력
2. 이메일 미인증 계정은 로그인 차단
3. 정지/영구정지 계정은 로그인 차단 + 사유 및 해제일 안내
4. 성공 시 Access Token 발급 (메모리 저장)
5. Refresh Token은 HttpOnly Cookie에 저장
6. 메인 페이지로 이동

### 로그아웃
1. 로그아웃 버튼 클릭
2. Access Token 메모리에서 삭제
3. Refresh Token Cookie 만료 처리

### 프로필 조회/수정
1. 닉네임, 프로필 이미지 변경 가능
2. 비밀번호 변경은 현재 비밀번호 확인 후 가능
3. 내가 쓴 글 목록 조회 가능

### 회원 탈퇴
1. 현재 비밀번호 입력으로 본인 확인
2. 탈퇴 확인 문구 입력 ("탈퇴하겠습니다")
3. 탈퇴 처리 → soft delete (deletedAt 기록)
4. 작성한 글/댓글은 "탈퇴한 사용자"로 표시, 내용은 유지
5. 즉시 로그아웃 처리

## 상세 요구사항

### 회원가입
- 이메일: 형식 검증 필수
  - 중복 체크: deleted_at 여부 무관 전체 레코드 대상 (탈퇴 계정 이메일 재사용 불가)
- 비밀번호: 8자 이상, 영문+숫자 조합 필수
- 닉네임: 2~20자, 중복 불가, 특수문자 불가
  - 닉네임 중복 체크: deleted_at IS NULL인 활성 계정만 대상 (탈퇴 계정 닉네임은 재사용 가능)

### 이메일 인증
- 가입 후 24시간 내 인증 필수
- 미인증 시 로그인 차단
- 인증 링크 재발송 요청 가능 (5분에 1회 제한)
- 인증 완료 후 링크 즉시 만료

### 토큰
- Access Token 유효기간: 30분
- Refresh Token 유효기간: 7일
- Refresh Token 만료 시 재로그인 필요

### 프로필 이미지
- 최대 5MB
- jpg, png, webp만 허용
- 미설정 시 기본 이미지 표시

### 회원 탈퇴
- soft delete 처리 (실제 데이터 삭제 X)
- 탈퇴 후 동일 이메일로 재가입 영구 불가
- 탈퇴한 계정의 닉네임은 즉시 반환 (재사용 가능)

## API 목록

| 메서드 | 엔드포인트 | 설명 | 인증 |
|--------|-----------|------|------|
| POST | /api/v1/auth/signup | 회원가입 | 불필요 |
| POST | /api/v1/auth/login | 로그인 | 불필요 |
| POST | /api/v1/auth/logout | 로그아웃 | 필요 |
| POST | /api/v1/auth/refresh | 토큰 재발급 | 불필요 |
| POST | /api/v1/auth/verify-email | 이메일 인증 확인 | 불필요 |
| POST | /api/v1/auth/resend-verification | 인증 이메일 재발송 | 불필요 |
| GET | /api/v1/users/me | 내 정보 조회 | 필요 |
| PATCH | /api/v1/users/me | 내 정보 수정 | 필요 |
| PATCH | /api/v1/users/me/password | 비밀번호 변경 | 필요 |
| DELETE | /api/v1/users/me | 회원 탈퇴 | 필요 |
| GET | /api/v1/users/me/posts | 내가 쓴 글 목록 | 필요 |
| GET | /api/v1/users/{id} | 타인 프로필 조회 | 불필요 |

## 예외 처리

| 상황 | HTTP 상태 | 에러 코드 | 메시지 |
|------|-----------|-----------|--------|
| 이메일 중복 | 409 | USER_EMAIL_DUPLICATE | "이미 사용중인 이메일입니다" |
| 닉네임 중복 | 409 | USER_NICKNAME_DUPLICATE | "이미 사용중인 닉네임입니다" |
| 비밀번호 불일치 | 401 | USER_LOGIN_FAILED | "이메일 또는 비밀번호가 올바르지 않습니다" |
| 이메일 미인증 | 403 | USER_EMAIL_NOT_VERIFIED | "이메일 인증이 필요합니다" |
| 인증 링크 만료 | 400 | USER_VERIFY_TOKEN_EXPIRED | "만료된 인증 링크입니다" |
| 토큰 만료 | 401 | AUTH_TOKEN_EXPIRED | "토큰이 만료되었습니다" |
| 정지된 계정 | 403 | USER_SUSPENDED | "정지된 계정입니다. 해제일: {date}" |
| 탈퇴한 계정 | 403 | USER_DELETED | "탈퇴한 계정입니다" |
