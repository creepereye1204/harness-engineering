# vote.md — 추천/비추천

## 기능 설명
게시글에 추천/비추천, 댓글/대댓글에 추천을 할 수 있는 기능.
본인 글/댓글에는 투표 불가, 중복 투표 불가, 취소 가능.

## 투표 대상별 허용 타입

| 대상 | 추천(UP) | 비추천(DOWN) |
|------|----------|-------------|
| 게시글 | ✅ | ✅ |
| 댓글 | ✅ | ❌ |
| 대댓글 | ✅ | ❌ |

댓글/대댓글의 DOWN 투표는 서비스 레이어에서 차단.
DB votes 테이블은 target_type=COMMENT일 때 DOWN이면
VOTE_DOWN_NOT_ALLOWED 에러 반환.

## 사용자 시나리오

### 추천/비추천
1. 로그인 사용자가 추천 또는 비추천 버튼 클릭
2. 즉시 UI 반영 (낙관적 업데이트)
3. 이미 같은 버튼을 눌렀으면 → 취소 (토글)
4. 추천 상태에서 비추천 클릭 → 추천 취소 + 비추천 등록 (자동 전환)
5. 비로그인 사용자 클릭 시 → 로그인 유도

### 투표 현황 표시
1. 게시글 상세: 추천수 / 비추천수 모두 표시
2. 게시글 목록: 추천수만 표시
3. 댓글/대댓글: 추천수만 표시
4. 본인이 투표한 버튼은 활성화 상태로 표시

## 상세 요구사항

### 투표 제한
- 본인 게시글/댓글에 투표 불가
- 로그인 사용자만 투표 가능
- 동일 대상에 중복 투표 불가
- 추천 ↔ 비추천 전환 가능 (게시글만)

### 카운터 갱신
- 투표/취소 시 posts.upvote_count, posts.downvote_count 즉시 갱신
- 댓글 투표 시 comments.upvote_count 즉시 갱신
- 갱신은 벌크 업데이트 쿼리 사용

### 비추천 블라인드
- 비추천수 - 추천수 >= 10이면 게시글 블라인드 처리
  (posts.is_blinded = true)
- "추천수 미달로 블라인드 처리된 게시글입니다" 표시
- 클릭 시 내용 펼치기 가능

### 인기글 반영
- 추천수는 post.md의 인기글 점수 계산에 반영
- 추천/비추천 발생 시 popularity_score 즉시 재계산

## API 목록

| 메서드 | 엔드포인트 | 설명 | 인증 |
|--------|-----------|------|------|
| POST | /api/v1/posts/{postId}/vote | 게시글 추천/비추천 | 필요 |
| DELETE | /api/v1/posts/{postId}/vote | 게시글 투표 취소 | 필요 |
| POST | /api/v1/posts/{postId}/comments/{commentId}/vote | 댓글 추천 | 필요 |
| DELETE | /api/v1/posts/{postId}/comments/{commentId}/vote | 댓글 추천 취소 | 필요 |

### 요청 바디 (게시글 투표)
```json
{
  "voteType": "UP"
}
```

## 예외 처리

| 상황 | HTTP 상태 | 에러 코드 | 메시지 |
|------|-----------|-----------|--------|
| 본인 글 투표 | 403 | VOTE_SELF_NOT_ALLOWED | "본인 게시글에는 투표할 수 없습니다" |
| 본인 댓글 투표 | 403 | VOTE_SELF_NOT_ALLOWED | "본인 댓글에는 투표할 수 없습니다" |
| 댓글 비추천 시도 | 400 | VOTE_DOWN_NOT_ALLOWED | "댓글은 비추천할 수 없습니다" |
| 비로그인 투표 | 401 | AUTH_REQUIRED | "로그인이 필요합니다" |
| 존재하지 않는 대상 | 404 | VOTE_TARGET_NOT_FOUND | "존재하지 않는 게시글/댓글입니다" |
| 투표 내역 없음 | 400 | VOTE_NOT_FOUND | "투표 내역이 없습니다" |
